🔧 Technical Requirements
1. Firebase Setup

Project: StudyVerse (Spark Plan - Free Forever).
Authentication:

Enable Google, Apple, Microsoft, Email/Password.
OAuth Consent Screen: Configure for thestudyverse.pages.dev.

Firestore:

Collection: users (documents keyed by user.uid).
Security Rules: Allow read/write for authenticated users only.
javascript
Copy

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}





Config: Hardcoded in app.js (replace placeholders with actual Firebase config).
2. Code Architecture

State Management:

Central state object in app.js tracks:

Current view (dashboard/favorites/settings).
Current tool (if any).
User data (nickname, favorites, etc.).
List of tools (modular: add/remove by editing state.tools array).


Modular Tools:

Each tool is defined in state.tools with:
javascript
Copy

{
  id: string,          // Unique ID (e.g., 'flashcards')
  name: string,        // Display name
  description: string, // Short description
  icon: string,        // Emoji or SVG
  type: string,        // 'built-in', 'iframe', 'external', 'mock-api'
  modalContent: string,// For built-in tools (e.g., 'flashcards')
  url: string,         // For iframe/external tools
  color: string        // Accent color (e.g., 'blue', 'violet')
}





Dynamic Rendering:

App grid and favorites grid render from state.tools and state.favorites.
Modals open dynamically based on tool type.

Copy
